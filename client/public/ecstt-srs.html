01 · OVERVIEW

نظرة عامة على النظام System Overview

الهدف: نظام ECSTT هو نظام تقييم النضج المؤسسي الشامل للمنظمات غير الربحية والجمعيات. يقيس النظام كفاءة المنظمة عبر 10 محاور مؤسسية باستخدام مقياس من 1 إلى 5، ويُصدر تقريراً تشخيصياً شاملاً مع توصيات مسار المعالجة.

Purpose: ECSTT is a comprehensive institutional maturity assessment system for non-profit organizations. It measures organizational effectiveness across 10 axes using a 1–5 scale and produces a full diagnostic report with treatment path recommendations.

المكوّن · Component التفاصيل · Detail
Base API URL https://gold-weasel-489740.hostingersite.com
بادئة المسارات · API Prefix /api/compliance/
محاور التقييم · Assessment Axes 10 محاور · 10 axes
مقياس التقييم · Score Scale 1 – 5 (integer or decimal per question)
تخزين الجلسة · Session Storage Submission ID stored in browser localStorage
المصادقة · Auth لا يتطلب توثيق مستخدم · No user authentication required
نوع المحتوى · Content-Type application/json
ملاحظة للمطوّر · Developer Note

جميع حسابات مسار المعالجة (AXIS_META) والأولويات الخمس وخارطة 90 يوم هي بيانات ثابتة حالياً في الواجهة الأمامية. الهدف من هذه الوثيقة هو تحويلها إلى استجابات API ديناميكية مبنية على نتائج التقييم الفعلية.

Currently, treatment path data (AXIS_META), Top 5 Priorities, and the 90-Day Roadmap are static constants in the frontend. The goal of this document is to migrate these into dynamic API responses based on actual assessment scores.

02 · FLOW

تدفق التقييم Assessment Flow

STEP-BY-STEP USER & API FLOW

1
بدء جلسة التقييم — يُرسل المستخدم طلب POST لبدء جلسة جديدة ويستلم submission_id + قائمة المحاور.
User triggers POST to initiate a new submission session and receives submission_id + axes list.
2
تحميل أسئلة كل محور — لكل محور (axis_id 1–10) يُرسل GET لتحميل أسئلته الخاصة.
For each axis (axis_id 1–10), a GET fetches that axis's questions.
3
المستخدم يُجيب على كل سؤال — يختار درجة من 1 إلى 5 لكل سؤال في المحور الحالي.
User selects a score (1–5) for each question in the current axis.
4
تسليم إجابات المحور — يُرسل POST بمصفوفة الإجابات لهذا المحور ثم ينتقل للتالي.
POST submits the answers array for the current axis, then advances to the next.
5
تكرار الخطوتين 2–4 لكل المحاور العشرة.
Repeat steps 2–4 for all 10 axes.
6
التقرير النهائي — بعد اكتمال جميع المحاور يُرسل GET لتحميل التقرير الكامل.
After all axes are complete, GET fetches the full final report.

مخطط التدفق · Flow Diagram

// Sequence: Frontend → Backend [User Opens /ecstt] ↓ POST /api/compliance/submissions/initiate ↓ ← { submission_id, axis_breakdown[] } ↓ FOR EACH axis (1 → 10): GET /api/compliance/axes/{axisId}/questions ← { axis_title, questions[] } [User answers each question] POST /api/compliance/submissions/{submissionId}/axes/{axisId}/answers ← { success: true } ↓ GET /api/compliance/submissions/{submissionId}/final-report ← { submission{}, organization{}, completion{}, axis_breakdown[], recommendations[] } ↓ [Redirect to /final-report/{submissionId}]
03 · API ENDPOINTS

نقاط الـ API API Endpoints

POST /api/compliance/submissions/initiate

الغرض: بدء جلسة تقييم جديدة وإرجاع معرّف التقديم مع بنية المحاور.

Purpose: Initiate a new assessment session and return submission ID with axis structure.

Request Body طلب

POST /api/compliance/submissions/initiate Content-Type: application/json // Body can be empty {} or contain optional org info { "organization_id": string | null, // optional "organization_name": string | null // optional }

Response استجابة

{ "submission_id": "abc-123", // REQUIRED — unique session token "submission": { "id": "abc-123", // alias for submission_id "status": "in_progress" }, "axis_breakdown": [ // array of axes { "axis_id": 1, "order": 1, "title": "الحوكمة والامتثال النظامي", "questions": [] // can be empty here; loaded separately } // ... 10 items total ] }
تنبيه · Important

الواجهة الأمامية تبحث عن submission_id في عدة مستويات من الاستجابة. يمكن إرجاعه في أي من المسارات: submission_id / submissionId / id / submission.id / data.submission_id

The frontend searches for the ID across multiple response levels. Accept any of: submission_id / submissionId / id / submission.id / data.submission_id

GET /api/compliance/axes/{axisId}/questions

الغرض: تحميل أسئلة محور معين. axisId من 1 إلى 10.

Purpose: Load questions for a specific axis. axisId ranges 1–10.

URL Parameters معاملات المسار

ParameterTypeRequiredDescription
axisIdinteger (1–10)مطلوبرقم المحور · Axis number

Response استجابة

// Format A — flat (preferred): { "axis_title": "القيادة والاستراتيجية", "questions": [ { "question_id": 1, "title": "وضوح الرؤية والرسالة والقيم المؤسسية", "weight": 1 } // ... more questions ] } // Format B — nested (also accepted): { "data": { "axis_id": 2, "title": "القيادة والاستراتيجية", "questions": [...] } }
حقول السؤال المقبولة · Accepted Question Fields

الواجهة تقبل أي من هذه الحقول لمعرّف السؤال: question_id / questionId / id

ونص السؤال من: title / question_title / question / question_text / text / label

POST /api/compliance/submissions/{submissionId}/axes/{axisId}/answers

الغرض: تسليم إجابات محور كامل.

Purpose: Submit all answers for a specific axis.

Request Body طلب

{ "answers": [ { "question_id": 1, // integer or string "score": 4, // integer 1–5 "notes": "" // optional string }, { "question_id": 2, "score": 3, "notes": "ملاحظة إضافية" } // one entry per question in the axis ] }

Response استجابة

// Success (any 2xx) { "success": true, "message": "Answers saved", "axis_score": 3.6 // optional — axis average computed by backend } // Error { "success": false, "message": "Submission not found" }
GET /api/compliance/submissions/{submissionId}/final-report

الغرض: استرجاع التقرير الكامل بعد اكتمال جميع المحاور.

Purpose: Retrieve the complete final report after all axes are submitted.

تفاصيل بنية الاستجابة موجودة في القسم 11.

Full response structure is detailed in Section 11.

حساب متوسط المحور · Axis Average Calculation

// Frontend calculation (before backend returns axis_score): function getAxisAverage(questions: ComplianceQuestion[], scores: Record<string, number>): number { const answered = questions.filter(q => scores[getId(q)] !== undefined); if (answered.length === 0) return 0; const sum = answered.reduce((total, q) => total + scores[getId(q)], 0); return Math.round((sum / answered.length) * 100) / 100; // Result: 2-decimal average e.g. 3.40 } // Backend should compute the same: // axis_score = SUM(scores) / COUNT(answered_questions) // overall_score = SUM(all axis_scores) / 10

08 · TOP 5 PRIORITIES

الأولويات الخمس المقترحة Top 5 Recommended Priorities

حالياً هذه القائمة ثابتة في الواجهة الأمامية. المطلوب من الـ API إرجاعها ضمن حقل top_priorities في استجابة التقرير النهائي، محسوبةً ديناميكياً بناءً على أدنى درجات المحاور.

Currently static in frontend. API should return these dynamically in top_priorities field of the final report, computed based on the lowest-scoring axes.

منطق الحساب الديناميكي · Dynamic Calculation Logic

يجب ترتيب الأولويات تنازلياً حسب الفجوة (5 — axis_score). المحاور الأقل درجةً = أولوية أعلى. يمكن مزج عدة محاور ذات صلة في أولوية واحدة.

Sort by gap desc: (5 - axis_score). Lowest-scoring axes = highest priority. Multiple related axes can be merged into one priority item.

10 · REPORT PARAGRAPH

فقرة التقرير النهائي Report Summary Paragraph

هذه الفقرة تُعرض في القسم الأخير من التقرير. يمكن توليدها من الواجهة الأمامية بناءً على البيانات، أو إرجاعها مباشرةً من الـ API ضمن حقل report_paragraph.

This paragraph appears at the end of the report. It can be generated client-side from data, or returned directly from the API in report_paragraph.

قالب الفقرة (عربي) · Arabic Template

بناءً على نتائج تقييم نضج المنظومة المجتمعية لـ[organization.name]، والتي أظهرت متوسطاً عاماً قدره [overall_score] / 5 وتصنيفاً ضمن مستوى [compliance_level]، فإن المنظمة تحتاج إلى مسار معالجة مؤسسي متكامل يبدأ بإعادة بناء الأساس الاستراتيجي والتشغيلي، ثم تفعيل القياس، وتنظيم البيانات، وبناء القدرات.

القالب (إنجليزي) · English Template

Based on the institutional maturity assessment results for [organization.name], which yielded an overall average of [overall_score]/5 placing it at the [compliance_level] level, the organization requires a comprehensive institutional treatment path beginning with rebuilding the strategic and operational foundation, followed by activating measurement systems, organizing data, and building capacity.
